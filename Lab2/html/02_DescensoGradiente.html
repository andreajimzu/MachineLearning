Cost Function y Gradiente¶

Lo que se busca es que dado un x y una y encontrar una prediccin de y (y hat) tal que esa prediccin provenga de la prob de que y = 1 dado ese valor de x.

Queremos que las predicciones se queden lo ms cerca posible de los valores reales, por lo que quiero minimizar el error ( esto es igual a la diferencia entre el valor real y el error en valor absoluto). Para no tener que usar la funcin valor absoluto se utiliza la funcin cuadrtica (elevndolo al cuadrado), otra forma es utilizando logaritmos.

Todo logaritmo tiene una funcin de prdida, porque as puedes meter tus propias funciones de coste. Las funciones de coste se pueden disear de muchas formas:

Loss function es una funcin que nos da el coste asociado con el modelo y mide cmo de bien lo est haciendo nuestro modelo en el training data.

                       Loss error function = (y, )=(ylog( )+(1y)log(1)

Por ejemplo: voy a suponer que mi valor real (y) es un 0 y que la prediccin es 1, es decir que he acertado la prediccin. Siempre que mi prediccin sea 1 se me anula ylog(yhat), luego al hacer el log (1 -yhat) es el log(1-1) = log(0) y eso es igual a infinito, por lo que el coste se dispara. Luego el (1 y) se queda en 0 por lo que me anula todo. Por lo que si yo acierto el error es 0.

Quiero minimizar mi coste, por lo que necesito saber cual es mi siguiente step o theta. Como quiero minimizar la funcin de coste, quiero que est lo ms cercano a 0 posible. Con la derivada parcial se puede encontrar el Theta para mi siguiente iteracin.

El gradiente es una medida que mide cmo de pronunciada est la pendiente. Tcnicamente cuando sumamos todas las primeras derivadas de todas las variables en una funcin, nos da el gradiente.

La funcin es la siguiente:

def costFunction(theta, X, y): # Coge theta, X e y y me devuelve la regresion logistica de la funcion coste y el gradiente 
   
    m=len(y)
    
    predictions = sigmoid(np.dot(X,theta)) # Pongo np.dot ya que X y theta son vectores.
    # Predictions corresponde con  de la frmula Loss error function.
    error = (-y * np.log(predictions)) - ((1-y)*np.log(1-predictions)) #Esta es la formula de Loss error function.

    cost = 1/m * sum(error) # Esta es la funcion de coste 
    
    grad = 1/m * np.dot(X.transpose(),(predictions - y)) # Esta es la funcion de gradiente 
    
    return cost[0] , grad
  • theta es un vector de coeficientes.
  • X es el vector de fila.
  • Y es un vector.

Se ha creado la funcin costFunction de tal forma que me calcule el coste para un determinado X e y.

La normalizacin consiste en transformar los datos de tal forma que todos los predictores estn aproximadamente en la misma escala.

Si normalizamos la magnitud del gradiente en cada step o paso del descenso de gradiente, entonces la longitud de cada step es igual al valor de nuestra tasa de aprendizaje (). Debido a esto, la tasa de aprendizaje se ajusta a s misma en cada step basado en la magnitud del gradiente, para asegurar que la longitud de cada step es .

La funcin de normalizacin que se utiliza es la siguiente:

def featureNormalization(X): #Coge un array de X valores y me devuelve valores normalizados de X, 
    # la media y la desviacion estandar de cada uno. 
   
    mean=np.mean(X,axis=0) #media de x 
    std=np.std(X,axis=0) #Desviacion estandar de x 
    
    X_norm = (X - mean)/std #Funcion de la normalizacion
    
    return X_norm , mean , std

Función de descenso de gradiente¶

Para optimizar parmetros necesitamos disminuir los errores. El objetivo del descenso de gradiente es alcanzar cada mnimo local. Se consigue mediante un proceso iterativo calculando el tamao del step para cada iteracin. Este clculo iterativo del tamao del step para alcanzar el mnimo local es conocido como descenso.

La frmula de descenso de gradiente es la siguiente:

Donde representa la tasa de aprendizaje. Se puede ver que influye bastante en el tamao del step ya que se multiplica el gradiente por el alpha para cada iteracin.

Lo que diferencia el descenso de gradiente de una regresin lineal es que en descenso de gradiente la hiptesis es ahora la funcin sigmoide.

def gradientDescent(X,y,theta,alpha,num_iters): #Coge un array de X, y, theta, alpha y num_iters, de tal forma
    # que se actualice theta cogiendo num_iters con la tasa de aprendizaje de alpha. 
    # Me devuelve theta y la lista del coste de theta durante cada iteracion.
   
    
    m=len(y) #m es la longitud de y
    J_history =[] # para que este entre corchetes el resultado
    
    for i in range(num_iters): #Hago un for para el rango del numero de iteraciones de tal forma que
        # el coste y el gradiente sea la funcion de coste de theta, X e y.
        cost, grad = costFunction(theta,X,y)
        theta = theta - (alpha * grad) #Formula para sacar el theta 
        J_history.append(cost) #hago un append para aadir en J_history los valores que salgan en cost 
    
    return theta , J_history #Theta y lista del coste de theta durante cada iteracion.
